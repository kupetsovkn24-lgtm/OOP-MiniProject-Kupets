# User Guide

## Призначення

`CarWorkshop` демонструє просту роботу адміністратора СТО через консоль: створення ремонту, перегляд результату, скасування запису і роботу з JSON-файлами.

## Запуск

```powershell
dotnet run --project src\CarWorkshop.Console\CarWorkshop.Console.csproj
```

## Меню

| Команда | Дія |
| --- | --- |
| `1` | Запустити повний сценарій ремонту й зберегти результат |
| `2` | Показати завантажені роботи |
| `3` | Створити і відразу скасувати новий запис |
| `4` | Показати запити, статистику і звіт дорогих ремонтів |
| `5` | Зберегти поточний стан у JSON |
| `6` | Завантажити стан із JSON |
| `0` | Завершити роботу |

## Основний сценарій

Для першого показу використовуйте порожню runtime-папку `data`, щоб застосунок створив демонстраційні записи з нуля.

1. Оберіть `1`.
2. Застосунок створить клієнта, автомобіль і механіка.
3. Буде створено та підтверджено запис на ремонт.
4. Застосунок створить роботу, призначить механіка та завершить ремонт.
5. На екрані з'явиться вартість і шлях до JSON-даних.
6. Оберіть `4`, щоб показати завершену роботу, виручку та звіт ремонтів від `4000 UAH`.

## Негативний сценарій

Команда `3` показує дозволене скасування нового запису. У коді й тестах також перевіряється, що не можна:

- створити запис у минулому;
- призначити зайнятого механіка;
- скасувати запис після створення роботи;
- змінити вже завершену роботу.

## Дані та помилки

JSON-файли створюються у runtime-папці `data` біля зібраної console application. Якщо JSON пошкоджено, застосунок повідомить про проблему і не підмінить стан частково завантаженими даними.
