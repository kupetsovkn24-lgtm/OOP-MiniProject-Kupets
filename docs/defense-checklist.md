# Defense Checklist - IW29

## Головний сценарій

1. Запустити застосунок і натиснути `1` для створення завершеного ремонту `4700 UAH`.
2. Натиснути `4` і показати статистику та новий звіт дорогих завершених ремонтів від `4000 UAH`.
3. Пояснити, що звіт використовує delegate-фільтр і LINQ-агрегацію.
4. Натиснути `5`, `6`, `2`, щоб показати збереження і відновлення.

## Негативний сценарій

Натиснути `3` після основного сценарію та показати контрольоване скасування нового запису. Додатково пояснити, що тести перевіряють: незавершена робота не потрапляє у звіт, а від'ємний поріг звіту відхиляється.

## Питання про архітектуру

| Питання | Коротка відповідь |
| --- | --- |
| Де видно SRP? | `Domain` тримає правила, `Application` сценарії і звіти, `Infrastructure` JSON, `Console` лише вивід. |
| Де видно OCP? | Нові pricing strategies і нові умови report-фільтра додаються без зміни `Job`. |
| Де видно DIP? | Use cases та facade працюють через repository interfaces. |
| Чому звіт у `Application`, а не в `Console`? | Це приклад прикладної логіки, Console тільки показує результат. |
| Чому залишився `Facade`? | Меню отримує невеликий зрозумілий API замість роботи з усіма use cases і queries напряму. |

## Питання про тестування

| Питання | Коротка відповідь |
| --- | --- |
| Який тест найважливіший для нового звіту? | Тест report-агрегації: він перевіряє фільтр, кількість і виручку. |
| Що перевіряють integration tests? | Реальне JSON save/load і помилки роботи з файлами. |
| Навіщо перевіряти порожній результат? | Щоб звіт коректно повертав нулі, коли дорогих ремонтів немає. |
| Навіщо тест від'ємного порога? | Щоб console/API не приймали беззмістовний параметр звіту. |
| Чому coverage недостатньо? | Він показує виконані рядки, але правильність доводять конкретні сценарії та assertions. |

## Питання про компроміси й альтернативи

| Питання | Коротка відповідь |
| --- | --- |
| Чому не додав базу даних? | Для навчального обсягу JSON уже покриває persistence, а БД збільшила б scope. |
| Чому фільтр через `Func<Job, bool>`, а не окремий складний клас? | Умова проста, delegate залишає код читабельним і дозволяє легко змінити критерій. |
| Що зробив би далі? | Додав би параметри звіту з меню або сховище з запитами для більшої кількості даних. |
