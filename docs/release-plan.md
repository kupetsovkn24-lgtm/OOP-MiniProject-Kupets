# Release Plan - v1.0.0

## Що входить у реліз

- Консольна система `CarWorkshop` для навчального сценарію СТО.
- Створення і підтвердження запису, створення роботи, призначення механіка, старт і завершення ремонту.
- Розрахунок вартості через `IPricingStrategy`.
- Скасування запису до створення роботи.
- JSON save/load із контрольованою реакцією на пошкоджені дані.
- Запити по роботах, завершених ремонтах і виручці.
- Unit та integration tests, coverage і GitHub Actions CI.
- Документація і сценарій демонстрації для захисту.

## Поза релізом

- База даних, веб-інтерфейс, авторизація.
- Облік запчастин як окрема складська підсистема.
- Резервні копії та транзакційне збереження кількох JSON-файлів.
- Реальні повідомлення клієнту.

## Допустимий технічний борг

- Дані зберігаються у JSON-файлах, бо це достатньо для навчального console application.
- Demo-сценарій створює приклад даних у меню, а не має окремого імпортера.
- Складний retry не додано: повторне читання пошкодженого JSON не усуває причину.

## Покриття тем курсу

- Добре покрито: ООП, інтерфейси, generics, LINQ, винятки, JSON, SOLID, патерни, UML, тести.
- Частково покрито: продуктивність розглянута на рівні доречності структур даних для малого набору.
- Свідомо не додається: GUI, БД і додаткові патерни лише заради кількості.

## Релізна умова

Після успішного merge Lab 37 у `main` і проходження CI репозиторій готовий до створення tag `v1.0.0`. Tag і GitHub Release створюються тільки після окремого підтвердження.
