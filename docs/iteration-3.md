## Усунені проблеми

- Помилковий JSON тепер може бути зафіксований через перевірюваний callback журналювання.
- Перевірено, що невдале завантаження не затирає вже наявний in-memory стан.
- Повторні збереження перевіряються окремим інтеграційним сценарієм.

## Політика відмови

- Відсутні файли трактуються як перший запуск із порожнім станом.
- Пошкоджені або суперечливі файли викликають `InvalidDataException`.
- Console показує повідомлення користувачу, а persistence може записати діагностичне повідомлення.
- Retry не додається: повторне читання пошкодженого файлу не усуває причину помилки.

## Ризики перед Lab 37

- Дані все ще зберігаються у кількох JSON-файлах без транзакції.
- Немає резервної копії файлів при перезаписі.
- Console залишається навчальним UI, а не повноцінною системою для СТО.
