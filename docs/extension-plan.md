# Extension Plan - IW29

## Мета

Додати простий аналітичний звіт по дорогих завершених ремонтах і посилити використання делегатів та LINQ без перебудови архітектури.

## Ланцюжок розширень

| Крок | Розширення | Вхідний стан | Результат для наступного кроку |
| --- | --- | --- | --- |
| A | Гнучкий фільтр завершених робіт | існуючі `Job` і `JobQueryExtensions` | метод приймає `Func<Job, bool>` та відбирає завершені роботи за умовою |
| B | Аналітичний звіт | результат фільтра A | `CompletedJobsReport` із переліком робіт, кількістю та загальною виручкою |
| C | Перевірка й показ | звіт B | facade/Console показують звіт, unit tests перевіряють відбір та підсумки |

## Мінімальний scope

- У звіті показуються завершені ремонти із сумою від `4000 UAH`.
- Звіт виводиться у вже наявній команді `4. Показати запити та статистику`.
- Доменні правила, JSON-формат і база даних не змінюються.

## Критерії готовності

- Звіт показує демо-ремонт `4700 UAH`.
- Незавершені або дешевші ремонти не входять у звіт.
- Порожній вибір повертає нульову кількість і суму.
- `dotnet build` і `dotnet test` проходять успішно.
